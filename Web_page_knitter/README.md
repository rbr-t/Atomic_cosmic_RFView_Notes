# Web-page-knitter

## Overview
Web-page-knitter is an R Shiny application designed to generate web pages from files organized in a folder hierarchy. The app supports various file types, including images, Word documents, PowerPoint presentations, Excel spreadsheets, and PDF files. It provides users with the ability to create individual reports for specific folders and a master report that aggregates these individual reports into a cohesive, linkable web page.

## Project Structure
The project consists of the following key files and directories:

- **app.R**: Contains the main logic and UI for the R Shiny app, managing server and UI components, file handling, and app behavior.
- **Report_generator_ShinyApp.Rmd**: RMarkdown template for generating individual reports, iterating through the specified source folder path and rendering an HTML page with embedded files in a hierarchical manner.
- **Master_html_report_ShinyApp.Rmd**: RMarkdown template for creating a master report that aggregates individual reports into a single, linkable web page.
- **www/**: Directory for web-accessible files.
  - **styles.css**: Custom CSS styles for the web pages generated by the app.
  - **bootstrapMint.css**: Additional styling based on a Bootswatch theme to enhance the appearance of the web pages.
  - **logo.png**: Default logo image used in the web pages generated by the app.
- **.github/copilot-instructions.md**: Instructions and guidelines for the development and usage of the Web-page-knitter app.

## Setup Instructions
To run the Web-page-knitter app locally, follow these steps:

1. Ensure that R and the required packages are installed on your system.
2. Open the `app.R` file in RStudio or any compatible IDE.
3. Run the app to start the Shiny server.

## Usage
- Navigate to the **Individual Reports** tab to generate reports for specific folders.
- Use the **Master Report** tab to create an aggregated report that links to individual reports.
- Customize the reports by specifying titles, authors, and logos as needed.
