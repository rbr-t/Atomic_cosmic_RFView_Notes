Path Handling Functions (5)

1. normalize_path_for_app(path)

Purpose: Convert any path format to usable absolute path

Handles:

  • Relative paths: IFX_2022_2025/01_Administration/
  • Absolute paths: /workspaces/project/IFX_2022_2025/
  • Windows paths: C:/Users/username/project/
  • Tilde paths: ~/Documents/project/
  • iShare URLs: https://apps.eu.valeo.com/ishare/...

Logic:

IF path is NULL or empty â†’ RETURN NULL
ELSE IF path starts with http/https â†’ convert_ishare_path()
ELSE IF path starts with ~ â†’ expand.path()
ELSE IF path is absolute (starts with /) â†’ normalize
ELSE (relative) â†’ join with workspace root â†’ normalize
FINALLY â†’ ensure trailing slash

2. normalize_cross_platform_path(path)

Purpose: Handle Windows/Unix path differences

Conversions:

  • Backslash â†’ Forward slash: C:\Users\... â†’ C:/Users/...
  • Remove drive letter: C:/Users/... â†’ /Users/...
  • Normalize separators: path1\path2 â†’ path1/path2

3. convert_ishare_path(path)

Purpose: Convert iShare URLs to local file paths

Input: https://apps.eu.valeo.com/ishare/sites/...

Output:

list(
  original_path = "https://apps.eu.valeo.com/...",
  converted_path = "/workspaces/.../path"
)

Mapping:

  • /sites/DE-TS-RFEngineering/... â†’ /workspaces/.../
  • Other paths â†’ Extract folder structure from URL

Path Validation Functions (2)

4. validate_source_path(path)

Purpose: Comprehensive path validation with detailed errors

Checks:

  1. Path is not NULL or empty
  2. Path exists on file system
  3. Path is a directory (not a file)
  4. Path is readable
  5. Directory contains files

Returns:

list(
  valid = TRUE/FALSE,
  error = "Error message" or NULL,
  suggestion = "Helpful suggestion" or NULL
)

Example Errors:

  • Non-existent: "Source path does not exist: {path}"
    Suggestion: "Check for typos. Available folders: ..."
  • Not a directory: "Source path is a file, not a directory: {path}"
    Suggestion: "Use parent folder instead"
  • Not readable: "Source path exists but is not readable: {path}"
    Suggestion: "Check permissions: chmod +r {path}"
  • Empty: "Source folder is empty: {path}"
    Suggestion: "Ensure folder contains files before rendering"

  2. Add documentation comment above function

  4. Return consistent data structure

Example:

# Calculate report size in MB
calculate_report_size <- function(report_path) {
  tryCatch({
    if (!file.exists(report_path)) {
      return(list(size_mb = 0, error = "File not found"))
    }
    size_bytes <- file.size(report_path)
    size_mb <- round(size_bytes / (1024^2), 2)
    return(list(size_mb = size_mb, error = NULL))
  }, error = function(e) {
    return(list(size_mb = 0, error = e$message))
  })
}

observeEvent(input$validate_all, {
  sections <- rv$sections
  if (length(sections) == 0) {
    showNotification("No sections to validate", type = "warning")
    return()
  }
  
  tryCatch({
    valid_count <- 0
    for (section_id in names(sections)) {
      section <- sections[[section_id]]
      validation <- validate_source_path(section$source_path)
      if (validation$valid) {
        valid_count <- valid_count + 1
      }
    }
    
    showNotification(
      sprintf("Validated %d/%d sections", valid_count, length(sections)),
      type = if (valid_count == length(sections)) "message" else "warning"
    )
  }, error = function(e) {
    showNotification(paste("Validation error:", e$message), type = "error")
  })
})

Performance Optimization

Reducing App Load Time

  • Minimize library loading (only load what's needed)
  • Lazy load modules (source inside functions if possible)
  • Cache expensive computations with memoise package

Improving Render Speed

  • Use knitr::opts_chunk$set(cache = TRUE) in .Rmd templates
  • Parallelize rendering with future package (advanced)
  • Reduce image sizes in source folders
  • Minimize number of sections rendered at once

  • Compress images before uploading logos
  • Minify CSS files (use online tools)
  • Clean up old rendered reports periodically
  • Use relative paths to reduce redundant copies
